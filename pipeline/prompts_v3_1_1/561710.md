## What you author

Write one JSON object conforming exactly to `pipeline/build/schemas/research_draft_v3_1_1.schema.json`. You author reusable atomic `evidence_facts` and selected research/judgment inputs. You do **not** author final provenance, deterministic dataset values, role weights, role contributions, factor scores, `S`, uplift or a verdict. Plain Python owns those fields.

Every selected input has a `method`:

- `OBSERVED` — `value` repeats a value stated by a cited fact exactly. Include the identical `observed_value`. No interpretation, proxy, normalization or judgmental bridge is permitted.
- `CALCULATED` — `value` is fully reproducible arithmetic over named numeric operands, every operand is backed by a cited fact, and the expression uses only numeric constants, parentheses and `+ - * /`. Include every operand's `name`, `value` and `fact_id`. Python recomputes the result. Any judgmental mapping makes this method invalid.
- `ESTIMATE` — required whenever any bridge from facts to the selected value is judgmental, including proxy choice, population mapping, normalization assumptions or an opinion. Its structured `estimate_basis` must enumerate every starting fact ID, every scope mismatch, each ordered proxy/normalization/population step, and the explicit judgmental step reaching the selected value. A cited background fact does not turn an estimate into `OBSERVED` or `CALCULATED`.

Python maps `OBSERVED → DIRECT`, `CALCULATED → DERIVED`, and `ESTIMATE → ESTIMATE`. Never author `provenance_type` in the draft. `evidence_quality` is a separate reliability axis: `HIGH | MED | LOW | NONE`. Empty `fact_ids` requires `NONE`; non-empty `fact_ids` requires non-`NONE` quality.

## Citation discipline — C1 to C4

1. **C1 — Fetch before cite.** Open and inspect the exact page before creating a fact. Search snippets, titles and memory are not evidence.
2. **C2 — Quote what you saw.** Each fact contains one short exact contiguous quote or table figure. Compact (`...`), spaced (`. . .`) and Unicode (`…`) ellipses are forbidden. Never splice separated passages.
3. **C3 — Attribute only what the URL establishes.** Never use a URL as support for a claim it does not establish. If no fetched source supports a background fact, omit the fact and label the selected value honestly as an estimate.
4. **C4 — Reopen and verify.** Before submission, enumerate and reopen every URL and verify its quote, attribution and all structured scope fields against the live page.

`evidence_facts[]` is the only place URLs may appear. Use stable unique IDs `F1`, `F2`, ... and reuse a fact by ID instead of duplicating its URL/quote. Every scope must state `population`, `geography`, `unit`, `period` and `industry_mismatch`. If a source omits any scope element, write exactly `not reported` for that field; never infer it silently.

## Frozen research rules

1. PE/M&A activity informs only B and M, never V or A.
2. Multiples are EV/EBITDA after market-rate owner compensation. State unit, target size, source population and every normalization. A normalization or market mapping that requires judgment is `ESTIMATE`.
3. A zero consolidator count is an absence-of-evidence estimate, not an observation. State the searches performed; do not cite a topical page as proving zero.
4. `current_adoption_pct` means the share of US firms in this NAICS materially using AI in automatable functions. Tool use, worker use and broad-sector use are different populations. Any judgmental mapping to the target population is `ESTIMATE`.
5. `t50_years` means years from today until 50% material firm adoption. If current adoption is already at least 50%, set it to `0`; otherwise it must be positive. Forecasting or analog-based bridging is normally `ESTIMATE`, even when supported by observed adoption facts.
6. Payer/regulatory repricing belongs in C when it determines who retains cost savings. Terminal class separately answers whether paid end demand survives mature AI; do not count the same clawback twice.
7. A statutory human-in-the-loop boundary affects terminal class only to the extent it preserves paid demand. It does not make automatable tasks non-automatable.
8. If a W-2 dataset labor share is distorted by 1099-heavy labor, use the injected value unchanged, flag `dataset_mismatch`, lower confidence as appropriate and show sensitivity. Do not silently adjust it.
9. Catch-all codes: identify at least the top three revenue subsegments, score the dominant subsegment, set `heterogeneous=true`, and complete the structured market-boundary disclosure.
10. `succession_shortage_documented` changes B. It is a full boolean selection under the same method/fact contract. `true` requires `OBSERVED`, at least one fact that directly documents a shortage, and exact live verification. A search-based `false` is normally `ESTIMATE` + `NONE`, with the searches and absence judgment fully disclosed in `estimate_basis`.

## Industry-specific guidance — hints, not facts

**Likely role structure:**
- **Pest control technicians (route service)** — ~55% of wage bill (OEWS basis: 37-2021 Pest Control Workers 54.7% of wage bill (60.3% of employment)); automatable: no. Licensed applicators physically treating structures on recurring routes — mature AI does not replace the visit. AI augments (route optimization, on-site pest ID from photos, service documentation) without removing the technician; smart traps/remote monitoring can reduce visit frequency on commercial accounts, which is a demand question, not a labor-substitution one.
- **Office, scheduling and customer service** — ~14-15% of wage bill (OEWS basis: 43-9061 Office Clerks 6.2% + 43-4051 Customer Service Representatives 2.9% + 43-6014 Secretaries/Admin 2.0% + 43-1011 Office Supervisors 2.1% + 43-3031 Bookkeeping Clerks 1.2%); automatable: yes. Scheduling, dispatch, billing, renewal processing and inbound-call handling — the genuinely automatable core; AI phone/scheduling agents apply directly here.
- **Management** — ~12% of wage bill (OEWS basis: 11-1021 General and Operations Managers 11.6%); automatable: partial. Multi-branch reporting, pricing and routing analytics automate; local hiring and commercial account relationships persist.
- **Sales** — ~6-7% of wage bill (OEWS basis: 41-3091 Sales Representatives of Services 6.5%); automatable: partial. Inspection-based selling (termite WDO reports) and door-to-door residential sales remain human; lead handling, quoting and follow-up automate.
- **Field supervisors** — ~4.5% of wage bill (OEWS basis: 37-1011 First-Line Supervisors of Housekeeping/Janitorial Workers 4.5%); automatable: no. Route supervision and training; AI widens span of control modestly.
- **Vegetation applicators and remainder** — ~7-8% remainder of wage bill (OEWS basis: 37-3012 Pesticide Handlers/Sprayers 1.8% + unlisted tail); automatable: no. Physical application work plus small misc tail.

**Suggested sources:**
- (adoption) PCT (Pest Control Technology) magazine State of the Industry report (annual) — Operator surveys covering technology adoption (routing software, AI phone agents, smart traps) and business metrics.
- (succession_owners) NPMA (National Pest Management Association) member/workforce data — Trade association covering the operator base; owner demographics and workforce shortage evidence.
- (consolidators_multiples) Potomac Pest Control Group M&A commentary (Paul Giannamore; 'The Buzz'/Potomac deal coverage) — The dominant sell-side advisor in pest control; regularly discusses prevailing EBITDA/revenue multiples for $1-10M targets and platform activity.
- (consolidators_multiples) PCT Top 100 list (annual) — Revenue rankings that quantify concentration and name the active acquirers (Rollins, Rentokil/Terminix, Anticimex, Certus and PE platforms).
- (other) Specialty Products Consultants 'Strategic Analysis of the U.S. Structural Pest Control Industry' (Gary Curl annual market survey) — The standard market-size and segment-mix (residential/commercial/termite) survey; confirm it is still published. *(existence unverified — confirm this source still exists/publishes before relying on it)*

**Capture questions:**
- Pricing is recurring per-visit/quarterly or annual subscription plans set locally, with no payer, auditor or fee schedule above the operator — when AI cuts office and routing costs, does the operator keep the spread as margin, or do national brands' door-to-door pricing and Google Local Services lead auctions compete it away in residential? Split the answer by residential recurring vs commercial contract vs termite/WDO work.
- Route density is the profit engine: the same AI scheduling gain is worth far more inside a dense route. In documented roll-up economics, how much of the margin uplift comes from densification and cross-selling versus back-office cost removal — and does that change what an AI-era acquirer should pay a standalone target?
- Who owns the customer: recurring residential accounts attach to the brand/phone number and transfer well in acquisitions (this is why the industry rolls up), but commercial accounts (food processing, healthcare, hospitality) hinge on compliance documentation and named relationships — what churn evidence exists post-acquisition for each?
- What could bypass the operator: smart-trap/remote-monitoring hardware reducing visit frequency on commercial contracts, DIY-plus-AI diagnosis at the residential low end, or pest-prevention-as-a-feature bundled by other home services — is there measurable revenue-per-account erosion anywhere yet?

**Terminal-value question:**
Pests are eliminated by a state-licensed technician physically treating the structure — mature AI does not perform the visit, so the product survives; the automatable surface is the ~30-35% of wage bill in office, sales, supervision and management around the route. The sharp question is therefore about capture and demand at the edges: does AI shift customer acquisition power to gatekeepers (Google LSA auctions, national ad-budget brands) and does remote monitoring thin out visit frequency on commercial accounts — leaving the local operator with the same truck rolls but weaker pricing? Distinguish AI cutting the operator's overhead (real, bounded) from any mechanism by which the paid visit itself goes away (largely absent here) — and keep the two out of a single blended score.

**Known traps:**
- EBITDA normalization: sub-$5M-revenue operators are owner-operated; quoted small-deal 'multiples' are frequently SDE multiples with owner comp added back — normalize to market-rate manager comp before comparing to platform exit multiples. Termite work adds retreat/repair guarantee liabilities (long-tail obligations) that quality-of-earnings must reserve for.
- n_band sensitivity: the chain's 15.65% generic margin puts only the 100-499 class (70 firms) in the $1-10M band, but well-run pest operators commonly run 20%+ EBITDA — at those margins part of the 20-99 class (766 firms, avg revenue $4.7M) enters the band. Treat n_band 70 as a floor and say so.
- Consolidation is intense and well documented (Rollins, Rentokil/Terminix, Anticimex, Certus, plus PE platforms; Potomac-brokered deals) — usable only for B (consolidator count) and M (multiples); never as evidence for V or A (rule 2).
- Seasonality and geography: mosquito/termite seasons make quarterly data and northern-vs-sunbelt comparisons misleading; sunbelt route density is structurally better.
- Recurring-revenue share is the valuation driver — one-time jobs vs recurring plans changes the multiple more than size does; verify the mix in any comp.
- Boundary: vegetation/lawn pest application overlaps 561730 (landscaping) codes; wildlife/animal removal and fumigation specialists sit inside 561710 but have different economics; state applicator licensing constrains technician supply (a labor-cost tailwind for AI-assisted retention/training, not a substitution channel).

## Frozen formulas — Python calculates them

- `V_raw = labor_share × ai_replaceable_share`
- `V = 10 × min(1, V_raw / 0.25)`
- `C = 10 × pi_dist × pi_moat`
- `A = 10` when `t50_years=0`, otherwise `min(10, 10 / t50_years)`
- `TD = clamp(log10(n_band) / 4, 0, 1)`
- `OW = clamp((owners_60plus_pct − 0.10) / 0.35, 0.1, 0.9)`; add 0.1 for documented succession shortage, cap at 1
- `CFD = min(0.9, log10(1 + active_consolidators) / 2)`
- `B = 10 × sqrt(TD × OW) / (1 + 0.3 × CFD)`
- `M = clamp(4 × (exit_mult − buy_mult) / buy_mult, 0, 10)`
- `S = (V × C × A × B × M)^(1/5)`
