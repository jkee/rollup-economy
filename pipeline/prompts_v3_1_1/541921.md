## What you author

Write one JSON object conforming exactly to `pipeline/build/schemas/research_draft_v3_1_1.schema.json`. You author reusable atomic `evidence_facts` and selected research/judgment inputs. You do **not** author final provenance, deterministic dataset values, role weights, role contributions, factor scores, `S`, uplift or a verdict. Plain Python owns those fields.

Every selected input has a `method`:

- `OBSERVED` — `value` repeats a value stated by a cited fact exactly. Include the identical `observed_value`. No interpretation, proxy, normalization or judgmental bridge is permitted.
- `CALCULATED` — `value` is fully reproducible arithmetic over named numeric operands, every operand is backed by a cited fact, and the expression uses only numeric constants, parentheses and `+ - * /`. Include every operand's `name`, `value` and `fact_id`. Python recomputes the result. Any judgmental mapping makes this method invalid.
- `ESTIMATE` — required whenever any bridge from facts to the selected value is judgmental, including proxy choice, population mapping, normalization assumptions or an opinion. Its structured `estimate_basis` must enumerate every starting fact ID, every scope mismatch, each ordered proxy/normalization/population step, and the explicit judgmental step reaching the selected value. A cited background fact does not turn an estimate into `OBSERVED` or `CALCULATED`.

Python maps `OBSERVED → DIRECT`, `CALCULATED → DERIVED`, and `ESTIMATE → ESTIMATE`. Never author `provenance_type` in the draft. `evidence_quality` is a separate reliability axis: `HIGH | MED | LOW | NONE`. Empty `fact_ids` requires `NONE`; non-empty `fact_ids` requires non-`NONE` quality.

## Citation discipline — C1 to C4

1. **C1 — Fetch before cite.** Open and inspect the exact page before creating a fact. Search snippets, titles and memory are not evidence.
2. **C2 — Quote what you saw.** Each fact contains one short exact contiguous quote or table figure. Compact (`...`), spaced (`. . .`) and Unicode (`…`) ellipses are forbidden. Never splice separated passages.
3. **C3 — Attribute only what the URL establishes.** Never use a URL as support for a claim it does not establish. If no fetched source supports a background fact, omit the fact and label the selected value honestly as an estimate.
4. **C4 — Reopen and verify.** Before submission, enumerate and reopen every URL and verify its quote, attribution and all structured scope fields against the live page.

`evidence_facts[]` is the only place URLs may appear. Use stable unique IDs `F1`, `F2`, ... and reuse a fact by ID instead of duplicating its URL/quote. Every scope must state `population`, `geography`, `unit`, `period` and `industry_mismatch`. If a source omits any scope element, write exactly `not reported` for that field; never infer it silently.

## Frozen research rules

1. PE/M&A activity informs only B and M, never V or A.
2. Multiples are EV/EBITDA after market-rate owner compensation. State unit, target size, source population and every normalization. A normalization or market mapping that requires judgment is `ESTIMATE`.
3. A zero consolidator count is an absence-of-evidence estimate, not an observation. State the searches performed; do not cite a topical page as proving zero.
4. `current_adoption_pct` means the share of US firms in this NAICS materially using AI in automatable functions. Tool use, worker use and broad-sector use are different populations. Any judgmental mapping to the target population is `ESTIMATE`.
5. `t50_years` means years from today until 50% material firm adoption. If current adoption is already at least 50%, set it to `0`; otherwise it must be positive. Forecasting or analog-based bridging is normally `ESTIMATE`, even when supported by observed adoption facts.
6. Payer/regulatory repricing belongs in C when it determines who retains cost savings. Terminal class separately answers whether paid end demand survives mature AI; do not count the same clawback twice.
7. A statutory human-in-the-loop boundary affects terminal class only to the extent it preserves paid demand. It does not make automatable tasks non-automatable.
8. If a W-2 dataset labor share is distorted by 1099-heavy labor, use the injected value unchanged, flag `dataset_mismatch`, lower confidence as appropriate and show sensitivity. Do not silently adjust it.
9. Catch-all codes: identify at least the top three revenue subsegments, score the dominant subsegment, set `heterogeneous=true`, and complete the structured market-boundary disclosure.
10. `succession_shortage_documented` changes B. It is a full boolean selection under the same method/fact contract. `true` requires `OBSERVED`, at least one fact that directly documents a shortage, and exact live verification. A search-based `false` is normally `ESTIMATE` + `NONE`, with the searches and absence judgment fully disclosed in `estimate_basis`.

## Industry-specific guidance — hints, not facts

**Likely role structure:**
- **Photographers (shooting + post-production)** — ~50% of wage bill (OEWS basis: 27-4021 Photographers 50.7% of wage bill (5-digit 54192 mix) — an unusually concentrated single-occupation industry); automatable: partial. The in-person capture (posing a family, running a school photo day, shooting a wedding) is physical and not automatable; but half or more of a portrait photographer's hours — culling, editing, retouching, album layout — is already substantially AI-automated (Imagen, Aftershoot class tools). Separately, AI image generation attacks the product itself (headshots from selfies); keep that in terminal value, not in the cost-side fraction.
- **Video/production staff** — ~5% of wage bill (OEWS basis: 27-4031 Camera Operators 2.6% + 27-2012 Producers and Directors 2.5%); automatable: partial. Event videography is physical; editing and highlight-reel production is heavily AI-automatable.
- **Management** — ~10% of wage bill (OEWS basis: 11-1021 General and Operations Managers 10.3%); automatable: partial. Scheduling, studio ops reporting and marketing automation are AI-tractable; school-district and league contract relationships persist.
- **Sales and retail (package/print selling)** — ~7% of wage bill (OEWS basis: 41-3091 Sales Reps of Services 3.0% + 41-2031 Retail Salespersons 1.9% + 43-4051 Customer Service Reps 1.9%); automatable: partial. Online proofing galleries and automated upsell flows replace in-person sales sessions; high-touch IPS (in-person sales) studios keep humans because the sales session IS the margin model.
- **Photo processing and production workers** — ~3% of wage bill (OEWS basis: 51-9151 Photographic Process Workers and Processing Machine Operators 3.1%); automatable: yes. Lab/print production and order fulfillment automate near-fully; most studios already outsource to consolidated labs.
- **Admin and back office** — ~17-20% of wage bill (OEWS basis: 43-9061 Office Clerks 4.3% + 43-3031 Bookkeeping Clerks 2.1% + unlisted admin tail est. ~10-15%); automatable: yes. Booking, order entry, invoicing, gallery delivery and customer comms automate near-fully.

**Suggested sources:**
- (adoption) PPA (Professional Photographers of America) benchmark surveys and member research — The main US trade association; publishes studio financial benchmarks and has surveyed members on AI tools and AI-generation threat.
- (adoption) Rangefinder / WPPI (Wedding & Portrait Photographers International) industry coverage — Trade coverage of AI editing-tool adoption (Aftershoot, Imagen) in wedding/portrait workflows — concrete adoption evidence.
- (capture_pricing) Shutterfly/Lifetouch corporate disclosures and school-photography contract coverage — Lifetouch (acquired by Shutterfly 2018) dominates contracted school/volume portraiture — the one consolidated, contract-locked sub-segment; its pricing model is the capture benchmark.
- (consolidators_multiples) BizBuySell Insight Reports (photography studio listings) — Small studios trade as SDE-multiple main-street deals; BizBuySell is the realistic comp source — there is no institutional multiple universe for this code.
- (succession_owners) PPA member demographic data — Owner-age and studio-closure data if published; the industry is 90% micro-firms so succession is mostly closure, not sale — verify what PPA actually reports. *(existence unverified — confirm this source still exists/publishes before relying on it)*

**Capture questions:**
- Portrait work is priced as session fees plus print/package upsells (or per-child package prices in contracted school photography) — when AI cuts editing and admin hours, does the solo-studio market's low entry barrier and price competition pass the savings to consumers immediately, or can branded studios hold session prices and pocket the margin?
- Who owns the client relationship — in volume photography it is the school district/league contract (institutional, renewable, owned by Lifetouch-class players); in retail portraiture it is personal to the photographer whose style the family chose. Which book is a roll-up actually buying, and does retail revenue survive the founder leaving?
- What stops a consumer from using a $20 AI headshot app or generating family portraits from phone photos instead of paying $200-500 for a session — and for which occasions (newborn, school, sports, weddings) does the physical event itself keep demand alive?
- In contracted school/volume photography, does the district RFP process claw AI savings back as lower per-package prices or higher commissions to the school, rather than leaving margin with the operator?

**Terminal-value question:**
This is a melter candidate: AI image generation attacks the product, not just the costs — a likeness no longer requires a photographer. The mechanism that decides survival is the physical occasion: school photo day, youth sports day, weddings and newborn sessions are events where presence, herding kids and the ritual itself are the product, while 'I need a headshot/portrait' demand is already migrating to generative apps. Score the terminal question segment by segment: how much of the code's revenue is occasion-locked physical events vs. output-only portraiture that generative AI replaces outright — and would a researcher notice that strong current session demand says nothing about the 5-year output-only segment?

**Known traps:**
- Melter + no targets: n_band is 21 firms — effectively zero roll-up universe. 8,742 of 9,682 firms are <5 employees with avg revenue ~$200K; the only firms in the $1-10M EBITDA band are the 100-499 class (21 firms). Any roll-up thesis here is buying micro-SDE businesses or competing with Shutterfly/Lifetouch in contracted volume work; score N and the overall opportunity accordingly.
- EBITDA normalization: studio 'profit' is almost always SDE with the owner-photographer's labor inside; market-rate replacement cost for the shooter typically erases most of the margin. Multiples are main-street SDE multiples (often <3x), not EBITDA multiples — do not import professional-services comps.
- Demand structure: the record's IPS-based labor share (0.2051, HIGH quality) is strikingly low for a personal-services code — consistent with heavy pass-through (prints, labs) and owner-proprietor labor not appearing in wages. Nonemployer firms vastly outnumber the 9,682 employer firms; SUSB-based counts understate the competitive fringe that sets prices.
- Boundary confusion: commercial photography is 541922 (separate code, different client base), photofinishing labs are 812921/812922, school-picture composite printing sits with the lab, and freelance photographers without employees are nonemployers outside SUSB — PPA membership and industry commentary span all of them; check segment before quoting.
- Sub-segment mix drives everything: contracted volume (school/sports/graduation — consolidated, contract-locked, Lifetouch-dominated), weddings/events (occasion-locked, fragmented), retail portrait studios (declining since the JCPenney/Sears studio collapse), and headshot/personal-branding work (most exposed to generative AI). State the mix explicitly before scoring V or terminal value.
- Golden-set note: this code anchors the melter end of the instrument — be conservative about anything that looks like a cost-savings windfall; the dominant effect is product substitution and it belongs in terminal value / pi_dist, not in a rosy automation margin.

## Frozen formulas — Python calculates them

- `V_raw = labor_share × ai_replaceable_share`
- `V = 10 × min(1, V_raw / 0.25)`
- `C = 10 × pi_dist × pi_moat`
- `A = 10` when `t50_years=0`, otherwise `min(10, 10 / t50_years)`
- `TD = clamp(log10(n_band) / 4, 0, 1)`
- `OW = clamp((owners_60plus_pct − 0.10) / 0.35, 0.1, 0.9)`; add 0.1 for documented succession shortage, cap at 1
- `CFD = min(0.9, log10(1 + active_consolidators) / 2)`
- `B = 10 × sqrt(TD × OW) / (1 + 0.3 × CFD)`
- `M = clamp(4 × (exit_mult − buy_mult) / buy_mult, 0, 10)`
- `S = (V × C × A × B × M)^(1/5)`
