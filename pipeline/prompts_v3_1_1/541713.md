## What you author

Write one JSON object conforming exactly to `pipeline/build/schemas/research_draft_v3_1_1.schema.json`. You author reusable atomic `evidence_facts` and selected research/judgment inputs. You do **not** author final provenance, deterministic dataset values, role weights, role contributions, factor scores, `S`, uplift or a verdict. Plain Python owns those fields.

Every selected input has a `method`:

- `OBSERVED` — `value` repeats a value stated by a cited fact exactly. Include the identical `observed_value`. No interpretation, proxy, normalization or judgmental bridge is permitted.
- `CALCULATED` — `value` is fully reproducible arithmetic over named numeric operands, every operand is backed by a cited fact, and the expression uses only numeric constants, parentheses and `+ - * /`. Include every operand's `name`, `value` and `fact_id`. Python recomputes the result. Any judgmental mapping makes this method invalid.
- `ESTIMATE` — required whenever any bridge from facts to the selected value is judgmental, including proxy choice, population mapping, normalization assumptions or an opinion. Its structured `estimate_basis` must enumerate every starting fact ID, every scope mismatch, each ordered proxy/normalization/population step, and the explicit judgmental step reaching the selected value. A cited background fact does not turn an estimate into `OBSERVED` or `CALCULATED`.

Python maps `OBSERVED → DIRECT`, `CALCULATED → DERIVED`, and `ESTIMATE → ESTIMATE`. Never author `provenance_type` in the draft. `evidence_quality` is a separate reliability axis: `HIGH | MED | LOW | NONE`. Empty `fact_ids` requires `NONE`; non-empty `fact_ids` requires non-`NONE` quality.

## Citation discipline — C1 to C4

1. **C1 — Fetch before cite.** Open and inspect the exact page before creating a fact. Search snippets, titles and memory are not evidence.
2. **C2 — Quote what you saw.** Each fact contains one short exact contiguous quote or table figure. Compact (`...`), spaced (`. . .`) and Unicode (`…`) ellipses are forbidden. Never splice separated passages.
3. **C3 — Attribute only what the URL establishes.** Never use a URL as support for a claim it does not establish. If no fetched source supports a background fact, omit the fact and label the selected value honestly as an estimate.
4. **C4 — Reopen and verify.** Before submission, enumerate and reopen every URL and verify its quote, attribution and all structured scope fields against the live page.

`evidence_facts[]` is the only place URLs may appear. Use stable unique IDs `F1`, `F2`, ... and reuse a fact by ID instead of duplicating its URL/quote. Every scope must state `population`, `geography`, `unit`, `period` and `industry_mismatch`. If a source omits any scope element, write exactly `not reported` for that field; never infer it silently.

## Frozen research rules

1. PE/M&A activity informs only B and M, never V or A.
2. Multiples are EV/EBITDA after market-rate owner compensation. State unit, target size, source population and every normalization. A normalization or market mapping that requires judgment is `ESTIMATE`.
3. A zero consolidator count is an absence-of-evidence estimate, not an observation. State the searches performed; do not cite a topical page as proving zero.
4. `current_adoption_pct` means the share of US firms in this NAICS materially using AI in automatable functions. Tool use, worker use and broad-sector use are different populations. Any judgmental mapping to the target population is `ESTIMATE`.
5. `t50_years` means years from today until 50% material firm adoption. If current adoption is already at least 50%, set it to `0`; otherwise it must be positive. Forecasting or analog-based bridging is normally `ESTIMATE`, even when supported by observed adoption facts.
6. Payer/regulatory repricing belongs in C when it determines who retains cost savings. Terminal class separately answers whether paid end demand survives mature AI; do not count the same clawback twice.
7. A statutory human-in-the-loop boundary affects terminal class only to the extent it preserves paid demand. It does not make automatable tasks non-automatable.
8. If a W-2 dataset labor share is distorted by 1099-heavy labor, use the injected value unchanged, flag `dataset_mismatch`, lower confidence as appropriate and show sensitivity. Do not silently adjust it.
9. Catch-all codes: identify at least the top three revenue subsegments, score the dominant subsegment, set `heterogeneous=true`, and complete the structured market-boundary disclosure.
10. `succession_shortage_documented` changes B. It is a full boolean selection under the same method/fact contract. `true` requires `OBSERVED`, at least one fact that directly documents a shortage, and exact live verification. A search-based `false` is normally `ESTIMATE` + `NONE`, with the searches and absence judgment fully disclosed in `estimate_basis`.

## Industry-specific guidance — hints, not facts

**Likely role structure:**
- **Research scientists (materials, chemistry, physics core)** — ~20-28% of wage bill (OEWS basis: role_mix is the 5-digit 54171 proxy, biased toward biotech: 19-1042 Medical Scientists 6.1% + 19-1021 Biochemists/Biophysicists 2.2% listed, but real nanotech firms skew to 19-2032 Materials Scientists, 19-2031 Chemists and 19-2012 Physicists in the unlisted tail, est. ~15-25%); automatable: partial. Plausible within-role fraction 0.2-0.4.
- **Engineers (mechanical, electrical, chemical, process)** — ~12-17% of wage bill (OEWS basis: 17-2141 Mechanical Engineers 1.9% listed + electrical/chemical/materials engineering occupations in the tail, est. ~10-15% more); automatable: partial. Design, modeling and documentation automate substantially; fab/cleanroom process work and equipment integration are physical.
- **Lab technicians** — ~8-11% of wage bill (OEWS basis: 19-4021 Biological Technicians 1.5% listed (proxy artifact) + chemical/engineering technicians in the tail, est. ~6-10%); automatable: no. Hands-on sample prep, instrument operation and cleanroom work; lab-automation-augmented but physically present.
- **Science and R&D management** — ~10-12% of wage bill (OEWS basis: 11-9121 Natural Sciences Managers 7.3% + 11-9199 Managers All Other 3.1%); automatable: partial. Sponsor/customer relationships and program direction persist; internal reporting and proposal assembly automate.
- **General management and operations** — ~9-10% of wage bill (OEWS basis: 11-1021 General and Operations Managers 5.2% + 13-1082 Project Management Specialists 2.6% + 13-1199 Business Operations Specialists 1.6%); automatable: partial. Grant/contract administration, compliance paperwork and project tracking automate heavily.
- **Software developers and computational staff** — ~7% of wage bill (OEWS basis: 15-1252 Software Developers 6.6%); automatable: partial. Simulation, instrument control and data pipelines; AI-augmented rather than eliminated.
- **Admin and remaining tail** — ~20-25% of wage bill (OEWS basis: not in listed top-10; 54171 top-10 covers only ~38% of wage bill — admin, facilities, QA and misc. occupations); automatable: partial. Admin/document/compliance work automates; facilities and EHS staff for labs/cleanrooms do not.

**Capture questions:**
- Contract R&D here is priced as cost-plus/cost-reimbursable government awards, fixed-price SBIR grants, or commercial NRE/FTE-rate agreements — under cost-plus and FTE-rate structures, AI-driven hour reductions flow to the sponsor, not the firm. Who keeps AI savings under each mechanism, and what is the revenue split across them for EBITDA-positive firms in this code?
- Who owns the customer relationship — the firm's contract vehicles and facility approvals, or individual principal investigators whose grants and reputations are personally portable?
- What stops sponsors (agencies, corporate R&D departments) from using AI plus their own labs to insource the analytical/design portion, leaving only physical characterization work outsourced?
- Is there recurring commercial revenue (materials supply, testing services, licensing) or is revenue one-shot grants and projects — and does AI shorten funded project scopes over time?

**Terminal-value question:**
The mechanism that holds paid demand here is physical: synthesis, fabrication and characterization require cleanrooms and instruments AI cannot replace, while AI-driven simulation and literature work shifts value away from the analytical hours firms bill. Does the dominant sub-segment's revenue sit in the physical-lab slice (durable) or the analysis/design slice (replaceable) — and does federally funded demand, which sets most budgets here, get recalibrated downward as agencies assume AI-accelerated research productivity?

**Known traps:**
- HETEROGENEOUS R&D CODE: 541713 spans nanomaterials developers, semiconductor-adjacent research shops, instrument-linked labs and grant-funded ventures. Identify the top-3 revenue sub-segments, score the DOMINANT one, and set heterogeneous: true. - DATA ODDITY — treat labor_share with suspicion: the record shows labor_share 0.0676 (QCEW 2024 wages $3.6B vs SUSB 2022 receipts $56.9B), implausibly low for an R&D services code and inconsistent with siblings (541714: 0.90, 541715: 0.49). Likely a classification mismatch — a few very large enterprises' receipts booked at this code in SUSB (note 232 firms in the 500+ class out of only 1,469 total) without matching QCEW wage records. Do not re-research the input (rule 6), but do not build V on it at face value; flag in top_uncertain_inputs.
- Business-model trap: many firms in this code are venture/grant-funded research ventures with no meaningful EBITDA — the $1-10M EBITDA band framing only fits the fee-for-service subset. The n_band of 301 (quality ESTIMATE, derived from a generic 15.65% services margin) likely overstates genuinely acquirable cash-flowing firms; probe what an EBITDA-positive nanotech 'business' actually is before scoring B.
- Pricing-model deflation candidate: federal cost-reimbursable awards and FTE-based commercial research agreements pass AI hour-savings to the sponsor — model in pi_dist with the mechanism stated.
- Boundary confusion: 541714 (biotech, incl. nanobiotech boundary disputes), 541715 (general physical/engineering R&D), 334xxx (semiconductor manufacturing, where firms with pilot production get reclassified) — market reports on 'nanotechnology' span products and services across all of these; check definitions before quoting.

## Frozen formulas — Python calculates them

- `V_raw = labor_share × ai_replaceable_share`
- `V = 10 × min(1, V_raw / 0.25)`
- `C = 10 × pi_dist × pi_moat`
- `A = 10` when `t50_years=0`, otherwise `min(10, 10 / t50_years)`
- `TD = clamp(log10(n_band) / 4, 0, 1)`
- `OW = clamp((owners_60plus_pct − 0.10) / 0.35, 0.1, 0.9)`; add 0.1 for documented succession shortage, cap at 1
- `CFD = min(0.9, log10(1 + active_consolidators) / 2)`
- `B = 10 × sqrt(TD × OW) / (1 + 0.3 × CFD)`
- `M = clamp(4 × (exit_mult − buy_mult) / buy_mult, 0, 10)`
- `S = (V × C × A × B × M)^(1/5)`
